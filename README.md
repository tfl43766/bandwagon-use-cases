## 搬瓦工能干什么：8 个真实用途

### 1. 建个人网站或博客

这是最经典的用法，也是大多数人入坑的起点。

买一台搬瓦工 VPS，装上宝塔面板，配好 WordPress，整个流程快的话两小时能跑通。和国内虚拟主机相比，VPS 的好处是不限带宽（在套餐月流量范围内）、不限数据库数量、不用备案就能上线。如果你只是想写文章、展示作品，入门款 KVM 套餐年付 $49.99 完全够用。

外贸网站也适合放在搬瓦工上。CN2 GIA-E 机房的套餐，国内访问快，海外访问也稳，对跑外贸电商来说是个实用选择。

### 2. 搭建私有网盘

不想把文件放百度、OneDrive，可以在 VPS 上跑 Nextcloud 或者 alist，自己搭一个私有网盘。文件上传下载速度取决于套餐带宽，CN2 GIA-E 套餐带宽够大，日常同步文档、照片问题不大。

这种用法有个实在的好处：文件不经过第三方服务器，隐私控制权在自己手里。

### 3. 学 Linux、练 Shell

买一台便宜 VPS 是学 Linux 最快的方式。本地虚拟机折腾起来麻烦，而且网络环境和真实服务器差很多。搬瓦工 KiwiVM 面板支持一键重装 20 多种系统，装坏了重装就行，不用担心搞乱什么。

我自己当年就是拿 $49.99 一年的套餐练手，Nginx 配置、SSH 密钥登录、防火墙规则，反复折腾没有负担。

### 4. 运行爬虫或定时任务

需要长期在后台跑脚本的场景，VPS 比本地电脑省电省心。常见的有：

- 价格监控爬虫（比如抢票、比价）
- 定时推送通知的小程序
- 数据采集脚本
- 定时备份任务

一台 1 核 1G 内存的搬瓦工 VPS 跑这类轻量任务完全够，不需要买高配。

### 5. 部署个人 API 服务

做开发的人经常需要一个公网可访问的后端。本地环境做内网穿透不稳定，而且 IP 会变。搬瓦工的固定公网 IP 解决这个问题，不管是跑 FastAPI、Node.js 还是 Go 写的服务，部署上去就能稳定跑着。

合租的朋友也可以考虑：一台 VPS 跑多个 Docker 容器，每个容器对应一个服务，互不干扰。

### 6. 自建 AI 工具和自动化工作流

这是近两年新起来的玩法。

n8n、Dify、OpenClaw 这类开源 AI 自动化工具，都可以部署在搬瓦工上。举个具体例子：n8n 接入 DeepSeek API，配好 Webhook，就可以实现自动整理邮件、定时生成报告、接收表单后触发 AI 处理——这些任务扔在自己的 VPS 上跑，不依赖第三方云服务，成本也低得多。

搬瓦工正在推进宿主机硬件升级，部分机房已经换到 AMD EPYC 处理器，跑 AI 推理和数据处理任务的性能比以前强不少。

不过要说明：本地运行大模型（比如 LLaMA 70B 这类）对内存和显存要求很高，普通 VPS 带不动，这类需求要上专用 GPU 服务器。搬瓦工的 VPS 适合跑调用 API 的 AI 应用，不适合本地跑大模型。

### 7. 搭游戏服务器

和朋友联机玩 Minecraft、Terraria、Valheim 这类游戏，在搬瓦工上跑一个专用服务器比用公网 IP 直连稳定得多。选机房时选离大家地理位置近的，延迟更低。

这种用法对带宽要求不高，但对延迟敏感。国内用户如果跑游戏服，优先考虑香港机房或日本机房，延迟低，体验好。

### 8. 挂机宝、DD Windows

挂机宝是指让 VPS 一直在后台保持某个程序运行，比如挂着 QQ、运行 Python 脚本、长期维持一个 SSH 连接等。

搬瓦工支持 DD 安装 Windows（通过 KiwiVM 挂载 ISO 镜像），装好之后可以用 RDP 远程桌面连进去，当一台永远不关机的 Windows 电脑用。
